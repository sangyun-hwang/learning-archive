# Google Cloud & Looker

## Cloud Computing

클라우드 컴퓨팅은 인터넷을 통해 컴퓨팅 리소스를 필요할 때 사용하는 방식입니다. 서버, 스토리지, 네트워크, 데이터 분석 도구를 직접 소유하지 않고 사용량 기반으로 활용할 수 있습니다.

장점은 다음과 같습니다.

- 접근성
- 확장성
- 비용 효율
- 관리형 서비스 활용
- 보안과 운영 부담 일부 감소

서비스 모델은 보통 IaaS, PaaS, SaaS로 구분합니다.

## Storage

클라우드 스토리지는 목적에 따라 다르게 선택합니다.

- File Storage: 파일 시스템 형태로 공유합니다.
- Object Storage: 이미지, 로그, 백업처럼 객체 단위로 저장합니다.
- Block Storage: VM 디스크처럼 블록 단위 저장소로 사용합니다.

## BigQuery

BigQuery는 Google Cloud의 관리형 데이터 웨어하우스입니다. 대용량 데이터를 SQL로 분석하고, 스토리지와 컴퓨팅을 분리해 빠르게 쿼리할 수 있습니다.

학습 포인트는 다음과 같습니다.

- SQL 기반 분석
- 예약 쿼리
- Dry Run을 통한 비용 예측
- 데이터 마이그레이션 고려 사항
- ETL과 ELT 흐름
- Batch ingestion과 Stream ingestion 차이

## 데이터 마이그레이션

마이그레이션 시에는 데이터 크기, 네트워크 대역폭, 다운타임, 보안 요구사항을 함께 고려해야 합니다.

5R 전략은 다음과 같습니다.

- Rehosting
- Replatforming
- Repurchasing
- Refactoring
- Retiring

## Looker

Looker는 데이터 분석과 시각화를 위한 플랫폼입니다. 데이터를 탐색하고 시각화하며, 대시보드를 공유해 의사결정에 활용할 수 있습니다.

데이터 분석 흐름은 보통 다음 순서로 진행합니다.

1. 질문 정의
2. 필요한 데이터 식별
3. 분석
4. 시각화
5. 해석과 공유

## 학습 상태

Google Cloud와 Looker는 데이터 분석 입문 관점에서 학습 중입니다. 프론트엔드 포트폴리오에서는 대시보드, 데이터 시각화, 클라우드 기반 분석 흐름을 이해한 보조 역량으로 정리합니다.
