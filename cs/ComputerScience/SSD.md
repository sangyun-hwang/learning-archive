# SSD 스케줄링과 내부 동작

## HDD와 SSD의 차이

HDD는 물리적인 플래터를 회전시키고 헤드를 이동시켜 데이터를 읽고 쓴다.

그래서 HDD에서는 다음 비용이 성능에 큰 영향을 준다.

- Seek Time: 헤드가 원하는 트랙으로 이동하는 시간
- Rotational Latency: 원하는 데이터가 헤드 아래로 올 때까지 기다리는 시간

HDD의 디스크 스케줄링은 결국 헤드 이동을 줄이는 것이 중요하다. FCFS, SSTF, SCAN, C-SCAN 같은 알고리즘은 요청 순서를 조정해 헤드 이동 비용을 줄이려는 방식이다.

SSD는 NAND Flash 기반 반도체 저장장치다.

- 헤드가 없다.
- 플래터 회전이 없다.
- 기계적인 이동 비용이 없다.

따라서 SSD에서는 HDD처럼 “헤드를 얼마나 적게 움직일까?”보다 “데이터를 어떻게 관리하고 병렬로 처리할까?”가 더 중요하다.

## SSD의 읽기, 쓰기, 삭제 단위

SSD를 이해할 때 중요한 점은 읽기, 쓰기, 삭제 단위가 다르다는 것이다.

- 읽기: page 단위
- 쓰기: page 단위
- 삭제: block 단위

page는 block보다 작은 단위다. 이미 데이터가 들어 있는 page에 바로 overwrite하기 어렵고, 다시 쓰려면 먼저 삭제가 필요하다. 그런데 삭제는 page 단위가 아니라 더 큰 block 단위로만 가능하다.

즉 한 page만 바꾸고 싶어도 같은 block 안의 다른 유효 데이터까지 함께 영향을 받을 수 있다. 이 제약 때문에 SSD는 기존 위치를 바로 고쳐 쓰기보다 새 위치에 데이터를 쓰고 기존 page를 invalid 처리한다.

## FTL

FTL은 Flash Translation Layer의 약자다.

운영체제는 저장장치를 논리 주소 기준으로 사용한다. 예를 들어 같은 파일의 같은 위치를 수정했다고 생각한다. 하지만 SSD 내부에서는 실제 물리 위치가 달라질 수 있다.

FTL은 다음 매핑을 관리한다.

```txt
논리 주소 <-> 실제 물리 주소
```

수정 요청이 오면 SSD는 보통 다음 방식으로 처리한다.

1. 새 page에 최신 데이터를 쓴다.
2. 논리 주소가 새 물리 주소를 가리키도록 매핑을 바꾼다.
3. 기존 page는 invalid 상태로 표시한다.

이 방식 덕분에 운영체제는 같은 위치를 수정한 것처럼 보지만, SSD 내부에서는 쓰기 제약에 맞게 다른 위치를 사용할 수 있다.

## Garbage Collection

SSD는 데이터를 새 위치에 계속 쓰기 때문에 invalid page가 쌓인다. 하지만 삭제는 block 단위로만 가능하므로, 단순히 invalid page만 골라 지울 수 없다.

Garbage Collection은 빈 block을 확보하기 위한 내부 정리 작업이다.

1. block 안의 유효한 page를 다른 block으로 복사한다.
2. 기존 block 전체를 삭제한다.
3. 새 쓰기에 사용할 수 있는 빈 block을 확보한다.

GC가 실행되는 동안에는 사용자 요청과 별개로 내부 읽기, 쓰기, 삭제가 추가로 발생한다. 그래서 쓰기 성능이 일시적으로 떨어질 수 있다.

## Write Amplification

Write Amplification은 사용자가 요청한 쓰기보다 SSD 내부에서 더 많은 쓰기가 발생하는 현상이다.

예를 들어 사용자는 작은 데이터 하나를 쓰려고 했지만, SSD 내부에서는 GC 때문에 유효 데이터를 다른 곳으로 옮기고 block을 지운 뒤 새 데이터를 쓰는 작업이 함께 발생할 수 있다.

```txt
사용자 쓰기 요청 < 실제 SSD 내부 쓰기량
```

Write Amplification이 커지면 성능 저하와 NAND 수명 감소로 이어질 수 있다.

## Wear Leveling

NAND Flash는 쓰기와 삭제 횟수에 제한이 있다. 특정 block만 반복해서 사용하면 해당 영역이 먼저 마모된다.

Wear Leveling은 쓰기와 삭제가 특정 영역에 몰리지 않도록 데이터를 여러 block에 고르게 분산하는 기술이다.

목적은 SSD 전체 수명을 균형 있게 사용하는 것이다.

## TRIM

TRIM은 운영체제가 SSD에게 “이 데이터는 더 이상 사용하지 않는다”고 알려주는 명령이다.

파일을 삭제해도 SSD는 그 물리 page가 더 이상 필요 없는지 바로 알기 어렵다. TRIM이 전달되면 SSD는 해당 page를 invalid로 처리해 이후 GC에서 더 효율적으로 정리할 수 있다.

## 병렬 처리

SSD는 여러 NAND Flash 칩과 채널을 동시에 사용할 수 있다. SSD 컨트롤러는 요청을 여러 채널에 분산해 병렬로 처리한다.

SSD의 성능 최적화는 HDD처럼 헤드 이동을 최소화하는 것이 아니라, 내부 병렬성, FTL 매핑, GC, Wear Leveling을 얼마나 효율적으로 관리하느냐에 달려 있다.
